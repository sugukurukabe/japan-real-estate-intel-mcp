# Release v8.0.0 — Authless directory readiness + artifact downloads

Directory-readiness pass: fixes every blocker found in an audit against the
Claude Connectors Directory / MCP spec review guidelines, plus artifact
portability and dashboard export UX improvements.

## Breaking changes

- **OAuth removed.** This server is now an authless public connector — no
  account, token, or client registration required for any tool. The previous
  OAuth 2.1 implementation had an auto-approving authorization endpoint and no
  real Dynamic Client Registration, which would not have passed directory
  review. Pro/Enterprise tiers are unlocked exclusively via ECDSA-signed
  license keys (`X-License-Key` header / `_licenseKey` tool argument),
  verified offline — never sent to Claude's own auth flow.
- `pdfBase64` / `xlsxBase64` fields removed from `GenerateReportOutput` /
  `ContractSupportOutput` / `ComparePrefecturesOutput` — superseded by
  `resource_link` artifact downloads (see below). If you parsed those base64
  fields directly, switch to fetching the returned `resource_link.uri`.

## Highlights

- **Downloadable artifacts** — `generate_area_report` (PDF), `compare_prefectures`
  (Excel), `portfolio_optimizer` (CSV), `generate_contract_support_package` and
  `assess_contract_risk` (Markdown) now persist their generated file to a
  TTL-cleaned store and return an MCP `resource_link` instead of embedding raw
  Base64. Served via `GET /artifacts/:id/:filename` (HTTP transport) or the
  `artifact://{id}` MCP resource (stdio transport).
- **Dashboard CSV/PNG export** — every tool-result widget card has an export
  toolbar using the official `@modelcontextprotocol/ext-apps` `app.downloadFile()`.
- **Full tool metadata** — `outputSchema` declared for all 38 tools,
  `idempotentHint` and a human-readable `title` on every tool, verified
  end-to-end in `tests/output_schemas.test.ts` / `tests/tool_annotations.test.ts`.
- **Free tier realigned** with the 3 demo prompts in
  [docs/free-demo-prompts.md](../free-demo-prompts.md), so a directory
  reviewer running the advertised examples doesn't hit a tier wall.

## Security fixes

- `getRequestTier()` no longer trusts an unsigned Base64 JSON `_licenseKey`
  payload — every key now goes through `verifyLicenseKeyOffline()`.
- `GET /api/license` now looks up a license by the opaque, single-use Stripe
  Checkout `session_id` instead of email, closing a license-key-disclosure-
  via-email-enumeration path.
- `GET /metrics` is gated by its own `METRICS_KEY` (or `API_KEY`), and 404s
  (rather than 401s) when no key is configured, so a public instance doesn't
  advertise the endpoint's existence.
- Free-tier monthly quota is now keyed by MCP session ID instead of client
  IP — avoids collapsing every anonymous user behind a hosted connector's
  shared egress IP into one quota bucket.

## Critical Docker fix

`better-sqlite3` never actually worked inside the production Docker image —
silent because `/health` still returned `200 ok` either way. Practical impact
before this fix: Free-tier monthly quota silently degraded to in-memory-only,
and PDF/Excel/CSV/Markdown artifact generation plus license-key storage/lookup
threw uncaught errors. Root cause and fix are documented in
[docs/deploy.md](../deploy.md#デプロイ後の動作確認sqlite依存機能) — anyone
running a pre-v8.0.0 image should rebuild.

## Upgrading

```bash
git pull
docker compose --profile <caddy|tunnel> up -d --build
```

Verify the SQLite fix landed correctly:

```bash
docker compose exec mcp node -e "
const { saveArtifact, getArtifact } = require('./dist/artifacts.js');
const m = saveArtifact('smoke test', 'smoke.txt', 'text/plain');
console.log(getArtifact(m.id)?.data.toString('utf-8'));
"
```
