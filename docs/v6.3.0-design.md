# v6.3.0 設計ドキュメント: ユーザー中心の完成形とMCP Apps UI

## 1. ターゲットユーザーペルソナ・ペインポイント・月額価値トリガー

### ペルソナ A: 不動産投資家（個人）
- **年齢**: 35-55歳
- **ペインポイント**: 複数サイトを巡回して地価・人口・リスクを手動で照合。1エリアの分析に2-3時間かかる
- **月額価値トリガー**: 「AIに聞くだけで、チャット内でインタラクティブなダッシュボードが出て、スライダーでシナリオ変えて、そのままPDFレポート出力できる」→ 時間を1/10に短縮
- **払える金額**: 3,000-8,000円/月（不動産取引1件の手数料と比較して安い）

### ペルソナ B: 法人（不動産仲介・デベロッパー）
- **ペインポイント**: 顧客への提案資料作成に毎回3日かかる。PLATEAU 3Dデータの活用方法がわからない
- **月額価値トリガー**: 「自社ブランドPDFレポートを自動生成。3D日影シミュレーションをチャット内で顧客に見せられる」
- **払える金額**: 10,000-30,000円/月（社員1人の半日分の人件費と比較）

### ペルソナ C: 店舗出店担当者（チェーン店本部）
- **ペインポイント**: 出店候補地の人流・競合・地価データを別々のツールで分析。意思決定が遅い
- **月額価値トリガー**: 「候補地を地図上でクリックするだけで出店スコアが出る。What-Ifで新駅開業後の予測まで見える」
- **払える金額**: 15,000-50,000円/月（出店判断の失敗リスクと比較）

### 共通の「逆転の発想」
従来: ユーザーがデータを探しに行く → AIに聞く → テキストで返ってくる → 自分で可視化
**逆転**: AIがリッチなインタラクティブUIを返す → ユーザーはUIを操作するだけ → 操作がサーバーツールを呼ぶ → AIが結果をコメント

## 2. 理想的なエンドトエンドフロー

```
ユーザー「名古屋駅周辺の投資分析を見せて」
  ↓
LLM → open_dashboard ツール呼び出し
  ↓
ホスト: _meta.ui.resourceUri から ui:// リソースをフェッチ
  ↓
チャット内にサンドボックスiframeでダッシュボードが表示
  ↓
ダッシュボード: AppBridge で初期データ（ツール結果）を受信・表示
  ↓
ユーザーがUI上で操作:
  ├── エリア変更（ドロップダウン）→ app.callServerTool('cross_analyze_real_estate_market', {...})
  ├── シナリオスライダー操作 → app.callServerTool('scenario_what_if', {...})
  ├── レイヤー切替（地価/リスク/人流）→ 表示切替（クライアントサイド）
  ├── 都道府県比較ボタン → app.callServerTool('compare_prefectures', {...})
  └── PDFエクスポートボタン → app.callServerTool('generate_area_report', {...})
  ↓
各ツール結果がUIにリアルタイム反映
  ↓
app.updateContext(): AIに対して「ユーザーは中村区に注目し、リニア開業シナリオで地価+18%を確認済み」と伝達
  ↓
ユーザー「この結果をレポートにして」→ AIが generate_area_report を自動呼び出し
```

## 3. MCPプリミティブへの必要機能マッピング

### Tool（サーバーが提供・モデルが制御）— 16ツール + 2 ChatGPT互換
| ツール | MCP Apps連携 | 用途 |
|--------|-------------|------|
| open_dashboard | `_meta.ui.resourceUri` でダッシュボードUI起動 | エントリーポイント |
| cross_analyze_real_estate_market | AppからcallServerToolで呼ばれる | エリア変更時のライブ更新 |
| scenario_what_if | スライダー操作で呼ばれる | リアルタイムWhat-If |
| forecast_land_price_trend | チャート更新で呼ばれる | 予測グラフ更新 |
| compare_prefectures | 比較ボタンで呼ばれる | スプリットビュー |
| generate_area_report | エクスポートボタンで呼ばれる | PDF/Markdown生成 |
| portfolio_optimizer | ドラッグ&ドロップで呼ばれる | ポートフォリオUI |
| simulate_aichi_future | 愛知シナリオパネルで呼ばれる | 将来価値シミュレーション |
| search / fetch | ChatGPT Apps SDK互換 | Deep Research連携 |
| その他8ツール | readOnlyHint: true | テキストベース分析 |

### Resource（サーバーが提供・アプリが制御）— 5リソース + MCP App UI
| リソース | URI | MCP Apps連携 |
|----------|-----|-------------|
| ダッシュボード2D | `ui://japan-real-estate-intel/dashboard` | MCP App HTML（AppBridge付き） |
| ダッシュボード3D | `ui://japan-real-estate-intel/dashboard-3d` | MCP App HTML（Three.js + AppBridge） |
| 地価データ | `realestate://land-price/{pref}/{area}` | データリソース |
| 浸水リスク | `hazard://flood/{pref}/{area}` | データリソース |
| 人口推移 | `stats://population-trend/{pref}/{area}` | データリソース |

### Prompt（サーバーが提供・ユーザーが制御）— 9プロンプト
| プロンプト | ガイドワークフロー |
|-----------|-------------------|
| investment_report | 投資判断レポート生成の手順テンプレート |
| store_location_evaluation | 店舗出店評価の手順テンプレート |
| prefecture_comparison | 都道府県横断比較の手順テンプレート |
| quick_start_examples | 初回ユーザー向けガイド |
| 他5プロンプト | 地価予測・シナリオ・ポートフォリオ等 |

### Completion（未使用 → v6.3.0で追加）
- エリア名のオートコンプリート（`completion/complete` でエリア名候補を返す）
- ツール引数の補完（都道府県名、用途タイプ）

### Logging（未使用 → v6.3.0で活用検討）
- ツール呼び出し回数・所要時間をMCPログとして送信
- 将来のティアリング判断データに活用

### Progress（未使用 → v6.3.0で追加）
- generate_area_report（PDF生成）の進捗通知
- 大量データのバッチ分析時の進捗表示

## 4. ティアリング設計（MCPプリミティブ対応）

### Free プラン
- **Tool**: search / fetch + 基本5ツール（cross_analyze, assess_risk, forecast, what_if, compare）
- **Resource**: データリソース（地価・浸水・人口）のみ
- **Prompt**: quick_start_examples のみ
- **制限**: 月50回のツール呼び出し、MCP Apps UIなし

### Pro プラン（月額 5,000円 想定）
- **Tool**: 全16ツール無制限
- **Resource**: MCP Apps ダッシュボード（2D + 3D）含む全リソース
- **Prompt**: 全9プロンプト
- **MCP Apps**: インタラクティブダッシュボード、双方向ツール呼び出し
- **特典**: ブランドPDFエクスポート、優先データ更新

### Enterprise プラン（月額 30,000円〜 想定）
- Pro全機能 + カスタムブランディング + API直接アクセス + SLA
